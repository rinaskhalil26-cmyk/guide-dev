# Workflow Git

* **branch** : Permet de créer un espace de travail isolé pour développer une nouvelle fonctionnalité sans toucher au code principal.
* **commit** : Sauvegarde une version du projet à un instant T avec un message descriptif.
* **push** : Envoie les commits locaux (de l'ordinateur) vers le dépôt distant (sur GitHub).
* **pull request** : Demande de fusionner le travail d'une branche vers une autre sur GitHub, permettant la relecture de code.
* **merge** : Fusionne concrètement l'historique de deux branches pour n'en former qu'un seul.
